📌 ¿Qué es TicketDesk?

Es un sistema web diseñado para que los empleados de una empresa puedan reportar problemas técnicos (computador no enciende, internet lento, etc.) y los técnicos de TI los resuelvan de forma organizada y eficiente.

⚙️ Backend - El "Cerebro" del Sistema

El backend es la parte que no se ve, pero hace todo el trabajo: guardar datos, procesar peticiones, ejecutar lógica de negocio.

📚 ¿Qué es Backend?

El Backend es la parte del software que corre en el servidor. Es como el motor de un carro: no lo ves, pero hace funcionar todo.

🐍 Lenguaje: Python

Python es uno de los lenguajes de programación más populares del mundo. Es fácil de leer, tiene muchas librerías y es usado por empresas como Google, Netflix e Instagram.

🛠️ Tecnologías Utilizadas

Tecnología¿Qué hace?
Flask Framework web ligero. Es la "estructura básica" sobre la que construimos la aplicación. Maneja URLs, peticiones y respuestas.

🗄️ Base de Datos - Donde se Guarda Todo

📚 ¿Qué es una Base de Datos?

Una Base de Datos (BD) es como un archivador digital gigante. Guarda toda la información del sistema: usuarios, tickets, mensajes, configuraciones, etc.

💾 Dos Opciones (según necesidad)

✅ SQLite (Desarrollo)

Para: Equipos pequeños (< 500 usuarios)

  • Sin instalación adicional
  • Funciona en un solo archivo
  • Perfecto para empezar
  • Soporta modo WAL (lectura/escritura concurrente)

🚀 PostgreSQL (Producción)

Para: Empresas grandes (500+ usuarios)

  • Soporta millones de registros
  • Múltiples conexiones simultáneas
  • Más rápido en escalas grandes
  • Estándar empresarial open-source

🔍 Búsqueda Ultra Rápida

📚 ¿Qué es FTS5?

FTS5 (Full-Text Search 5) es una tecnología de búsqueda dentro de la base de datos. Permite buscar entre millones de tickets en menos de 200 milisegundos (más rápido que parpadear). Funciona como Google pero dentro de la app.

🔒 Seguridad - Protección de Datos

🛡️ Capas de Seguridad

🔑
JWT con Blacklist

JSON Web Token: identificación digital del usuario. Si el usuario cierra sesión, el token se invalida en menos de 1 segundo.

🔐
PBKDF2 SHA-256

Algoritmo de hash para contraseñas: las cifra con 100,000 iteraciones. Imposible de revertir.

🏢
LDAP / Active Directory

Integración con el Directorio Activo corporativo. Login con las mismas credenciales de Windows.

📝
Audit Log

Registro de auditoría: cada acción importante queda registrada (quién, cuándo, qué hizo).

🚧
Multi-Tenant

Aislamiento total: usuarios de Eliot NO pueden ver datos de Pash o Primatela. Filtrado en cada consulta.

🧼
Sanitización

Limpieza de entradas para prevenir XSS (scripts maliciosos) e inyección SQL.

📚 Glosario de Seguridad
  • XSS (Cross-Site Scripting): Ataque donde alguien intenta inyectar código malicioso en una web.
  • Inyección SQL: Ataque donde se intenta manipular la base de datos a través de formularios.
  • Hash: Función que transforma una contraseña en un texto irreversible.
  • Token: Tipo de "pase digital" que identifica al usuario sin enviar la contraseña.
  • RBAC: Control de acceso basado en roles (Admin, Técnico, Empleado).

🤖 Inteligencia Artificial Integrada

El sistema tiene un bot inteligente que puede responder preguntas automáticamente y un sistema de 4 agentes que procesan los tickets.

💬 Bot de Soporte Híbrido

Usuario pregunta ↓ 1. Buscar en Base de Conocimiento (KB) - 56 soluciones precargadas - Algoritmo de coincidencias por palabras clave ↓ 2. Si NO hay match → Llamar a Claude (IA de Anthropic) ↓ 3. Si Claude falla → Respuesta genérica + asignar especialista
📚 ¿Qué es Claude?

Claude es una Inteligencia Artificial creada por la empresa Anthropic. Es similar a ChatGPT pero con enfoque en seguridad y precisión. El bot usa Claude Haiku 4.5, una versión rápida y económica.

🎯 4 Agentes Especializados (Orchestrator)

🏷️
1. Classifier

Clasifica automáticamente el ticket en una de 12 categorías (Hardware, Software, Red, etc.)

🎯
2. Assignor

Asigna el ticket al técnico ideal según sus habilidades y carga de trabajo actual.

💬
3. Responder

Genera la respuesta inicial al usuario consultando la base de conocimiento o llamando a Claude.

⚠️
4. Escalator

Proceso en segundo plano que cada 5 minutos detecta tickets con SLA en riesgo y los escala.

📚 ¿Qué es SLA?

SLA (Service Level Agreement) = "Acuerdo de Nivel de Servicio". Es el tiempo máximo que tiene un técnico para resolver un ticket. Ejemplo: tickets críticos = 2h, altos = 4h, medios = 8h, bajos = 24h.

🐳 Despliegue con Docker

📚 ¿Qué es Docker?

Docker es una herramienta que empaqueta la aplicación junto con todo lo que necesita (Python, librerías, base de datos) en "contenedores". Es como un envío con todo dentro: lo puedes ejecutar en cualquier computador y funciona igual.

📦 4 Contenedores

Contenedor¿Qué hace?Tecnología
app Ejecuta la aplicación Flask Python 3.10 + Flask
postgres Base de datos productiva PostgreSQL 15-alpine
redis Caché de sesiones y datos temporales rápidos Redis 7-alpine
nginx Proxy reverso: recibe peticiones y las dirige a la app + SSL Nginx 1.25-alpine

🛡️ Seguridad en Contenedores

no-new-privileges

Los contenedores no pueden ganar más permisos de los que tienen.

cap_drop: ALL

Se eliminan todos los permisos del sistema operativo por defecto.

read_only filesystem

Algunos contenedores son de "solo lectura": nada puede modificarlos.

Healthchecks

Cada 10-30 seg se verifica que el contenedor esté funcionando.

📚 Glosario Técnico Completo

Definiciones de los términos técnicos usados en el proyecto:

Conceptos Base
  • Backend: Parte del software que corre en el servidor. No se ve, hace el trabajo.
  • Frontend: Parte visible de la aplicación. Lo que ve y usa el usuario.
  • API: "Interfaz de programación". Conjunto de URLs que permiten al frontend hablar con el backend.
  • REST: Estilo arquitectónico para diseñar APIs (usa verbos HTTP: GET, POST, PUT, DELETE).
  • JSON: Formato de datos liviano usado para enviar información entre el frontend y backend.
  • CRUD: Las 4 operaciones básicas: Create, Read, Update, Delete (Crear, Leer, Actualizar, Borrar).
Base de Datos
  • SQL: Lenguaje estándar para hablar con bases de datos relacionales.
  • SQLite: Base de datos ligera que cabe en un solo archivo.
  • PostgreSQL: Base de datos robusta para entornos empresariales.
  • ORM: Permite usar código Python en lugar de SQL para hablar con la BD.
  • Migración: Cambios estructurales en la BD (agregar tabla, columna, etc.).
  • FTS5: Sistema de búsqueda de texto ultra rápido dentro de SQLite.
Web y Red
  • HTTP: Protocolo de comunicación entre navegador y servidor.
  • HTTPS: HTTP cifrado (con candado verde).
  • WebSocket: Conexión permanente que permite enviar mensajes instantáneos en ambos sentidos.
  • Endpoint: Una URL específica que el frontend puede llamar.
  • Proxy Reverso: Servidor que recibe peticiones y las redirige al servidor correcto.
  • SSL/TLS: Tecnología que cifra la comunicación HTTPS.
Seguridad
  • Autenticación: Verificar QUIÉN es el usuario (login).
  • Autorización: Verificar QUÉ puede hacer ese usuario (permisos).
  • JWT: Token de identificación que el cliente envía en cada petición.
  • Hash: Transformación irreversible de un texto (para contraseñas).
  • LDAP: Protocolo para conectarse al Directorio Activo de la empresa.
  • RBAC: Control de acceso por roles (Admin, Técnico, Empleado).
  • CORS: Sistema para controlar qué webs pueden hablar con tu API.
DevOps e Infraestructura
  • Docker: Tecnología de contenedores que empaqueta apps con sus dependencias.
  • Contenedor: Caja virtual aislada donde corre una aplicación.
  • Docker Compose: Herramienta para gestionar varios contenedores juntos.
  • CI/CD: Integración y Despliegue Continuos (automatizar tests y deploy).
  • Healthcheck: Verificación automática de que un servicio está vivo.
  • Logs: Registros que escribe el sistema con cada acción importante.
Conceptos Avanzados
  • Multi-Tenant: Una app que sirve a múltiples organizaciones con datos aislados.
  • Microservicios: Arquitectura donde la app se divide en servicios pequeños independientes.
  • Caché: Almacenamiento temporal de datos para acceder más rápido.
  • Optimistic Locking: Sistema para prevenir conflictos cuando 2 usuarios editan lo mismo.
  • Background Job: Tarea que corre en segundo plano (escalación, backups, etc.).
  • Webhook: URL que recibe notificaciones automáticas de otros sistemas.
  • SLA: Acuerdo de tiempos de respuesta. Ej: críticos = 2h.
