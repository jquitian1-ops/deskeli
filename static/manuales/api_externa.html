🔌 API Externa de DeskEli

Documentación para integraciones — crear tickets desde sistemas externos

Introducción

DeskEli expone una API REST para que sistemas externos (Aranda, RRHH, monitoreo, ITSM, etc.) puedan crear tickets y consultar su estado de forma programática.

Base URL https://deskeli.eliotproyectos.tech/api/v1/external

Autenticación

Todas las peticiones requieren una API Key como Bearer token en el header:

X-Authorization: Bearer dsk_t_XXXXXXXXXXXXXXXXXXXXXXXXX

Alternativamente se acepta el header estándar Authorization: Bearer ....

Cómo obtener una API Key

  1. Un administrador entra a /admin/config → 🔌 API / Integraciones
  2. Click en ➕ Generar nueva API Key
  3. Elige nombre, empresa, scopes y (opcional) fecha de expiración
  4. Copia el token que aparece — solo se muestra una vez
⚠️ Seguridad El token es un secreto. Guardalo en un gestor (Vault, AWS Secrets Manager, .env con permisos restrictivos). Nunca lo commitees a Git ni lo pongas en logs.

Scopes disponibles

ScopeDescripción
tickets:createCrear nuevos tickets
tickets:readConsultar el estado de tickets existentes

Endpoints

POST /api/v1/external/tickets — Crear ticket

Headers:

Content-Type: application/json
X-Authorization: Bearer dsk_t_TU_TOKEN

Body:

{
  "subject": "Servidor SAP no responde",
  "description": "El SAP PRD no responde desde hace 20 min...",
  "category": "SAP",
  "priority": "critical",

  // Identificar al solicitante (uno de los dos)
  "applicantEmail": "juan.perez@patprimo.com.co",
  // o bien:
  // "applicantId": 42,

  // Opcionales
  "authorId": 10,
  "assigneeEmail": "tecnico@patprimo.com.co",
  "userArea": "Contabilidad",
  "userLocation": "Piso 3, Bogotá",
  "userPhone": "3001234567",
  "externalRef": "SR-98765",

  // Campos personalizados (compatibilidad con Aranda)
  "listAdditionalField": [
    {"name": "Ubicación", "stringValue": "ELIOT"},
    {"name": "USUARIO AD", "boolValue": true}
  ]
}

Campos:

CampoTipoObligatorioDescripción
subjectstring✅Título del ticket (max 200 chars)
descriptionstring✅Descripción del caso (soporta markdown básico)
applicantEmail o applicantIdstring / int✅Identifica al usuario solicitante. Debe existir en DeskEli y pertenecer a la empresa del token.
categorystring—Categoría (default: "General")
prioritystring—low / medium / high / critical (default: medium)
authorIdint—Quién crea el ticket. Default: el applicant.
assigneeEmail / assigneeIdstring / int—Técnico asignado (rol technician o admin)
userArea, userLocation, userPhonestring—Contacto del solicitante
externalRefstring—ID del ticket en el sistema origen (se anexa a la descripción)
listAdditionalFieldarray—Campos personalizados (compat Aranda); se anexan a la descripción

Respuesta 201 (éxito):

{
  "success": true,
  "id": 42,
  "ticket_number": "TKT-ELIOT-00042",
  "url": "https://deskeli.eliotproyectos.tech/technician/ticket/42",
  "status": "open",
  "created_at": "2026-07-02T15:30:00",
  "sla_deadline": "2026-07-02T17:30:00"
}

Errores comunes:

CódigoCausa
400Faltan campos subject/description, o el applicant no existe
401Token faltante, inválido, revocado o expirado
403Token sin scope tickets:create, o applicant de otra empresa
500Error interno (ver logs de DeskEli)

Ejemplos completos

Python (equivalente al script de Aranda)

import requests

url = "https://deskeli.eliotproyectos.tech/api/v1/external/tickets"
token = "dsk_t_XXXXXXXXXXXXXXXXXXXXXXXXX"

headers = {
    "Content-Type": "application/json",
    "X-Authorization": f"Bearer {token}"
}

payload = {
    "subject": "Prueba de creación desde API",
    "description": "Ticket generado por integración externa.",
    "category": "General",
    "priority": "high",
    "applicantEmail": "juan.perez@patprimo.com.co",
    "userArea": "Contabilidad",
    "externalRef": "SR-98765",
    "listAdditionalField": [
        {"name": "Ubicación", "stringValue": "ELIOT"},
        {"name": "USUARIO AD", "boolValue": True}
    ]
}

response = requests.post(url, headers=headers, json=payload)

if response.status_code == 201:
    data = response.json()
    print(f"✅ Ticket creado: {data['ticket_number']}")
    print(f"   URL: {data['url']}")
else:
    print(f"❌ Error {response.status_code}: {response.json()}")

Migración desde Aranda

Compatibilidad con Aranda El endpoint acepta campos como listAdditionalField y externalRef para minimizar cambios en scripts existentes. Los campos personalizados se anexan a la descripción del ticket como metadata visible.

Mapeo de campos Aranda → DeskEli

ArandaDeskEliNotas
subjectsubjectigual
descriptiondescriptionigual
applicantIdapplicantId o applicantEmailDeskEli acepta email también
authorIdauthorIdigual
responsibleIdassigneeId / assigneeEmailrenombrado
categoryIdcategory (string)DeskEli usa nombre, no ID
priorityIdpriority (string)low/medium/high/critical
projectId, serviceId, modelId, stateId, impactId, urgencyId—DeskEli no los usa (modelo más simple)
listAdditionalFieldlistAdditionalFieldse anexa a description

Buenas prácticas

  • Un token por consumidor: creá una key distinta para Aranda, otra para RRHH, otra para monitoreo. Facilita revocar solo la que se comprometió.
  • Scopes mínimos: si un sistema solo crea, no le des tickets:read. Principio de mínimo privilegio.
  • Rotá los tokens: revocá y generá uno nuevo cada 6 meses. Los tokens no expiran por default.
  • Rate limit: el sistema aplica 120 req/min por IP. Si necesitás más, hablá con el admin.
  • Idempotencia: si mandás el mismo externalRef dos veces, se crean 2 tickets (no hay deduplicación). Manejalo del lado del cliente.
⚠️ Si sospechás que un token se filtró Andá inmediatamente a /admin/config → 🔌 API / Integraciones, pausá esa key o eliminala, y generá una nueva. Todo uso posterior de la key comprometida será rechazado.

Soporte

Para problemas de integración: crear un ticket en el sistema con categoría "Integración API" o contactar al administrador de DeskEli.
