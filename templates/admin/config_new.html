⏱ Tiempos de SLA por Prioridad

📅 Horario Laboral para SLA

Define qué días y horas cuentan para el SLA. Si está activo, el sistema solo consume tiempo de SLA durante el horario laboral; los fines de semana, feriados y horas fuera de jornada no descuentan tiempo. Esto evita que tickets creados un viernes a las 5pm aparezcan vencidos el lunes.

Si está desactivado, el SLA usa tiempo calendario (24/7). Si está activado, solo cuenta horas hábiles.

Activa cada día y define su horario específico. Ejemplo: Lun-Vie 08:00-18:00, Sáb 09:00-13:00, Dom cerrado.

Ej: 25/12 Navidad, 01/01 Año Nuevo, feriados nacionales.

💡 Ejemplo práctico:
Si el horario es Lun-Vie 08:00-18:00 y se crea un ticket el viernes 17:30 con SLA de 4 horas:
• Sin horario laboral: vence el viernes a las 21:30 (mientras nadie trabaja)
• Con horario laboral: vence el lunes a las 11:30 (30min restantes del viernes + 3:30h del lunes)

📥 Buzones de Correo (Email → Ticket)

Configura hasta 2 buzones IMAP. El sistema revisa estos correos cada N minutos y crea un ticket automáticamente por cada correo no leído.

💡 Servidores comunes:
• Gmail: imap.gmail.com:993 (SSL) — Password con App Password
• Office 365: outlook.office365.com:993 (SSL) — RECOMENDADO: OAuth 2.0
• Yahoo: imap.mail.yahoo.com:993 (SSL) — Password con App Password
🔐 Cómo configurar OAuth 2.0 para Microsoft 365 (click para expandir)
Microsoft está deshabilitando Basic Auth para IMAP. OAuth 2.0 es la única forma confiable a futuro.

Paso 1 — Registrar la app en Azure AD:
  1. Ir a portal.azure.com → Microsoft Entra ID → App registrations → New registration
  2. Nombre: DeskEli IMAP Reader
  3. Account types: Accounts in this organizational directory only
  4. Redirect URI: dejar vacío
  5. Click Register
  6. Copiar el Application (client) ID y el Directory (tenant) ID
Paso 2 — Crear secret:
  1. En la misma app → Certificates & secrets → New client secret
  2. Descripción: "DeskEli secret", expiración: 24 meses
  3. Copiar el "Value" inmediatamente (no se ve después)
Paso 3 — Asignar permisos API:
  1. API permissions → Add a permission → APIs my organization uses
  2. Buscar: Office 365 Exchange Online
  3. Application permissions → marcar IMAP.AccessAsApp
  4. Click Grant admin consent (requiere admin del tenant)
Paso 4 — Habilitar la app en el buzón (PowerShell): 
Connect-ExchangeOnline
New-ServicePrincipal -AppId "<CLIENT_ID>" -ServiceId "<OBJECT_ID_DE_APP_ENTERPRISE>"
Add-MailboxPermission -Identity "cat@patprimo.com.co" -User "<CLIENT_ID>" -AccessRights FullAccess
Set-CASMailbox -Identity cat@patprimo.com.co -ImapEnabled $true
Paso 5 — Pegar los 3 valores acá abajo: Tenant ID, Client ID, Client Secret.
⏰ Los cambios de Azure tardan 5-15 min en propagarse.
